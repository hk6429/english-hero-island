# 英語英雄島

國小三至六年級英語學習扶助遊戲化自學系統。學生透過短回合診斷、能力練功、Boss 戰與跨日修煉，建立可解釋的英語能力證據。

## 專案狀態

目前已完成可玩的「學生自主冒險垂直切片」、「教師快派／學生匿名加入／合作 Boss」課堂版程式垂直切片，以及支援真人雙人複核的題庫治理垂直切片，但不是正式公開版，也尚未通過第一階段內容閘門。學生端仍固定使用原本 60 題試作內容；題庫治理候選庫已擴充到 200 題原創草稿，但在兩位英語教師逐題完成複核前，新增的 140 題不會進入學生流程或正式課堂活動。

目前可操作的流程：

- 三、四、五、六年級皆可完成建立英雄、五題診斷、能力島、提示工具選擇、五題練功、Boss、結果與修煉排程。
- 60 題原創可玩試作題，每年級 15 題；另有 140 題只供治理流程匯入與真人複核，合計每年級 50 題、全庫 200 題草稿。
- 200 題治理匯入包會移除版本、狀態、作者與複核者等欄位，再由伺服器依登入身分建立治理紀錄；正解位置經穩定打散，避免固定第一格偏誤。
- 可產生 200 題教師草稿預審包：包含來源與逐題 SHA-256、教師閱讀 CSV、兩份獨立空白回覆表、49 題試作素材阻擋清單，以及 49 筆正式素材 intake template；它不冒充正式凍結、真人複核或密碼學簽章。
- 首次獨立答對、提示後答對、救援完成與待支援分開記錄。
- XP、護盾、連擊、島嶼修復、能力卡、圖鑑與跨日精熟規則已接上。
- 三種提示策略、兩條等值練功路線、三種英雄光環、受控 Boss 招式與結果故事分支已可操作。
- 任務單選群組支援方向鍵、Home／End 與 roving tabindex；提示與作答回饋出現後會接住鍵盤焦點。
- 聽力文字輔助預設隱藏並先警告可能含作答線索；自主冒險與課堂 TTS 試作中主動揭露後，該題只記為提示後完成，不冒充獨立聽力精熟。課堂輔助標記可跨答題前元件重建與離線待送保存，但仍是瀏覽器回報的教學 telemetry，不是防竄改評量證據。
- 完成主線後可進入星光秘境；每個不同學習日最多取得 1 把不會過期的星鑰，用來揭露一顆事前不洩漏內容的知識星。
- 可選擇 Web Share 鼓勵卡或同裝置真人策略接力；下一位學伴必須選擇一個具體應用方式再回傳，策略與回覆都收入本機收藏。兩者都不含姓名、XP、分數、速度或錯題。
- 進度以 IndexedDB 保存在學生目前的瀏覽器，不要求帳號或真實姓名。
- 題庫治理以核准身分、每人每版本唯一且不可變的複核紀錄取代可偽造的 JSON 名單；作者不能複核自己的版本。
- 兩位不同的有效英語教師通過後只會進入 `reviewed`，不會自動發布；正式發布、爭議回報與退役都是分開的管理動作並留下不可變事件。
- `/governance` 提供真人登入閘門、凍結版本預覽、正解／解析／提示／授權證據及七項品質檢核；研究參考資料不得直接發布成題目。
- 複核者送出前完全收不到其他教師的通過或退回票數；英語教師的一般題庫搜尋也只允許 `published`，完整版本票數只供內容編輯與管理員，避免繞道形成從眾偏誤。
- 音訊／圖片題送審前，受信任匯入程序必須讀取實際檔案，核對 SHA-256、byte length、magic-byte MIME、題目綁定與授權證明；題目快照會凍結這份 byte receipt metadata。複核者可在題卡與確認框查看完整題目及素材收據，發布時也會重算快照雜湊並比對目前證據。資料庫不會自行讀取物件 bytes，收據也不是數位簽章或法律意見，且尚待真實 PostgreSQL runtime 驗證。
- 核准內容編輯可建立單題草稿、原子批次匯入、搜尋與預覽全庫、送審及從最新版本建立修訂；批次匯入只需題號與題目內容，版本、狀態、作者與複核者都由伺服器建立。
- 核准管理員可在同一工作區執行發布、爭議回報與退役；核准英語教師另有已發布題目搜尋與爭議回報介面。版本比較會保留前後內容、資產／來源網址、修改摘要與線性版本關係。
- 同一題同時只能有一個正式發布版本；新版發布前必須先停用舊版。`tts:`、`scene:`、`data:` 等試作或內嵌素材均不得發布，正式音訊／圖片必須使用以檔案 SHA-256 命名的 content-addressed locator，且凍結證據與發布當下證據必須完全一致。
- 品質面板會檢查重複／多重答案鍵、必要音訊或圖片缺漏、可驗證的 TTS 文字不一致、授權、爭議與具足夠樣本的異常答對率；低樣本或無法確認的外部資產只標示資料不足，不假裝已判定內容錯誤。
- 教師可用 Magic Link 登入、建立與封存班級、管理匿名別名／學習代碼／小組，並建立 3／5 題課堂任務。
- 全班任務不需目標名單；小組至少選 2 位匿名學生，個別任務恰選 1 位。活動與目標名單以同班級外鍵綁定，不能只掛上裝飾性的「小組」標籤。
- 學生以六碼活動碼、匿名暱稱與選填的六碼學習代碼加入；小組或個別任務會由伺服器確認該代碼確實在目標名單，且不能被第二個匿名身分重複占用。單一匿名身分每 10 分鐘最多嘗試加入 12 次，失敗原因採一致回應。
- 教師可停止新加入、結束活動，並在重新整理後接回最近活動；已保存的學習事件不會因結束活動而刪除。
- 課堂作答會先以 IndexedDB 保存在目前裝置，再送交伺服器判分；斷線、重整或回應遺失時沿用同一事件 ID 自動重送。已提交事件即使活動隨後結束仍可原樣重播，不同內容的衝突重送會拒絕，成功判分前不顯示正解。
- 活動結束後，教師可查看不含姓名與個別答案的匿名彙總報告；未達 5 位參與者、六成完整作答與每題 3 份證據時，系統只標示資料不足，不推論共通弱點。
- 證據達門檻後，報告才標示可能的共通卡點，並依能力提供「短講示範、等值變式、隔日確認」三步補救建議。
- 課堂看板只顯示已加入、進行中、已完成與可能需要協助，不顯示分數、速度或排名。
- 全班作答會累積共同修復值與 Boss 護甲；學習事件不可直接修改或刪除。
- 未設定專用 Supabase 專案時，學生、教師與題庫治理入口都會停在安全閘門，不建立假的跨裝置活動或複核結果。

課堂與治理目前仍有明確邊界：資料庫 migration、RLS、RPC、題庫建立／匯入／搜尋／修訂／品質監測、素材 bytes 驗證器，以及複核／發布／爭議／退役前端流程的程式實作均已完成並通過靜態契約與元件測試，但尚未套用到專用 Supabase 專案，也尚未做真實 PostgreSQL、跨裝置或 49 份正式媒體的 runtime 驗證。現有帳號中的其他 Supabase 專案不會被借用。

完整完成條件包含：

- 國小三至六年級完整主線。
- 200 題經兩位英語教師複核的原創變式題。
- 學生冒險、教師課堂與題庫治理三個子系統。
- Octalysis 實測至少 70／80，並另行通過兒童安全閘門。
- 內容、資安、無障礙、多裝置及自動化測試全部有直接證據。

## 核心原則

- Boss 通關不等於能力精熟。
- 答錯不會 Game Over，也不會失去 XP、能力、徽章、提示或必學內容。
- 只有 Learning Engine 能建立不可變學習事件；戰鬥與獎勵只讀取事件。
- 自主使用不要求真實姓名、電子郵件、照片或生日。
- 未經兩位英語教師複核的題目不得標示為正式發布。

## 本機開發

需求：Node.js 22 LTS 與 npm。

```bash
nvm use
npm install
npm run dev
```

開啟 `http://localhost:3000`。

未提供 Supabase 公開設定時，自主冒險照常運作；`/join`、`/teacher` 與 `/governance` 會顯示安全閘門。

## 課堂後端設定

先建立一個只供本系統使用的 Supabase 專案，並確認方案與費用。不要連到既有的其他正式專案。接著在 Supabase Authentication 開啟匿名登入與教師電子郵件登入，才執行：

```bash
cp .env.example .env.local
npx supabase link --project-ref <專用專案 ref>
npx supabase db push --dry-run
npx supabase db push
```

再將專用專案的 URL 與 publishable key 填入 `.env.local`。瀏覽器端只接受公開金鑰；service role key 不得放進 `NEXT_PUBLIC_*` 變數。

Migration 不會自動灌入題庫或建立複核者。正式開放前仍須由管理流程建立核准教師、核准內容複核者、班級，以及完成雙人複核的正式題目。詳細信任邊界與上線閘門見 `docs/classroom-security.md` 與 `docs/question-governance.md`。

要重新產生 200 題的內容式批次匯入檔，可執行：

```bash
npm run export:review-candidates
```

輸出位於 `artifacts/question-bank/review-candidate-question-bank.json`。它是待治理草稿，不是已複核正式題庫；`tts:` 與 `scene:` 試作資產必須在發布前換成可驗證的正式資產。

要產生可交給兩位英語教師獨立閱讀與填寫的草稿預審包，可執行：

```bash
npm run export:teacher-prereview
```

輸出位於 `artifacts/teacher-prereview-pack/draft-<來源雜湊>/`。其中清楚標示正式發布阻擋，不包含或偽造教師身分、複核時間、票數、正式版本鎖定或簽章。

## 正式素材驗證

目前 200 題中有 151 題純文字題，以及尚待製作的 24 份音訊、25 份圖片；正式媒體實體檔案目前仍是 0／49。預審包內的 `production-assets.manifest.template.json` 已列出精確的 49 筆 intake，但所有 receipt 欄位保持空白，不會偽造完成狀態。

完成實體素材與去識別授權證明後，依 `artifacts/question-assets/README.md` 建立正式 manifest，再執行：

```bash
npm run ingest:production-assets
```

驗證器會直接讀取素材與權利文件 bytes，檢查 SHA-256、長度、magic-byte MIME、content-addressed locator、題號／素材種類及獨立權利組合。49 筆全部通過後才會輸出 production question bank 與 verification report；缺 manifest、缺檔或任一證據不符都會以退出碼 1 阻擋。匯入鎖涵蓋「舊報告失效、讀檔、驗證、暫存寫入、最後提交報告」全流程；同一工作目錄不可平行執行，iCloud 上的不同電腦也不得同時匯入。magic bytes 只確認檔頭，不代表媒體一定可播放或解碼。這個本機 CLI 尚未替專用 Supabase 建立題目版本或呼叫 `register_question_asset_evidence`，因此不能視為端到端正式匯入完成。

## 已驗證品質

- 381 個單元、契約與元件測試，涵蓋 81 個測試檔。
- 34 個 Playwright 流程：四個年級各在桌機與 360px 手機完成主線，另含鍵盤答錯救援、課堂／治理安全閘門，以及 18 個 axe 與鍵盤焦點無障礙流程。
- `lint`、TypeScript 型別檢查與 production build 全數通過。
- 360px 主流程沒有橫向捲動；選項高度、焦點樣式與 reduced motion 已納入設計系統。
- Octalysis 獨立程式證據複評為 70／80；分項、直接證據、扣分理由與兒少安全限制均記錄於稽核文件。

上述 70／80 是獨立審查代理針對目前程式證據的重評，不等於真人三角色正式驗收或正式發布。內容雙人複核、課堂版專用後端 runtime 驗證、遊戲設計／英語教學／兒童 UX 三角色中位數評分、兒童實測、資安與完整 WCAG 驗收仍是硬性閘門。
