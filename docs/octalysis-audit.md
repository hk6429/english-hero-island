# 英語英雄島 Octalysis 實作稽核

更新日期：2026-07-14

## 稽核定位

本文件依周郁凱（Yu-kai Chou）的 Octalysis 八角遊戲化框架，盤點「目前已實作且可操作」的證據。這是開發階段自評，不是正式驗收分數。

正式門檻維持：遊戲設計、英語教學、兒童 UX 三種角色各自審查，各驅力取三人中位數，合計至少 70／80。未接通的功能不以規格目標分代替實測分。

## 朝 80 分與正式驗收的必要修正

目前已達進入三角色評分的最低開發證據門檻。以下項目仍是正式版與滿分目標的必要工作：

1. 社交影響：在專用後端完成班級合作 Boss、教師看板與共同修復進度的跨裝置實測，再補教師預設鼓勵卡；不公開個人速度、錯題或排名。
2. 創意授權：加入組句卡、多解任務與學生可安全創作的例句收藏。
3. 史詩意義：讓班級共同修復進度推進跨週章節，而非只增加個人數字。
4. 所有權：增加不付費、不競逐稀有度的島嶼展示配置。
5. 實測：由遊戲設計、英語教學、兒童 UX 三種角色獨立評分，逐項保留畫面、行為與學生試玩證據；中位數總分未達 70 就繼續修正，不以平均分掩蓋弱項。
6. 安全：完成真實學生焦慮、羞恥、刷題副作用與家長／教師理解度測試；安全閘門未過，即使分數達 70 也不得發布。

## 兒童安全檢查

目前學生端已確認：

- 一般題沒有倒數。
- 答錯不顯示「失敗」「太慢」「你不會」或 Game Over。
- 不因錯題、缺席或中斷移除 XP、能力卡、提示、解析或必學內容。
- 沒有公開排行榜、付費優勢、限量名額、轉蛋或隱藏機率。
- 英雄光環與兩條路線只改變外觀或探索呈現，不改變題目難度、提示與 XP。
- 同桌鼓勵必須由真人接手裝置選擇，系統不捏造同學、教師或班級活動。
- 課堂版只呈現全班共同修復與支援狀態，不呈現個人分數、速度或排名；未連線時停在安全閘門。
- Boss 通關與能力精熟分開呈現。
- 試作題固定揭露「待兩位英語教師複核」。

尚未完成的真實學生焦慮、羞恥、刷題副作用與家長／教師理解度測試，仍屬正式發布阻擋項目。
