# 英語英雄島 Octalysis 實作稽核

更新日期：2026-07-14

## 稽核定位

本文件依周郁凱（Yu-kai Chou）的 [Octalysis 八角遊戲化框架](https://yukaichou.com/gamification-examples/octalysis-gamification-framework/)，盤點「目前已實作且可操作」的證據。本輪先由獨立程式審查代理按直接證據重評；這仍不是遊戲設計、英語教學與兒童 UX 真人三角色的正式驗收分數。

正式門檻維持：遊戲設計、英語教學、兒童 UX 三種角色各自審查，各驅力取三人中位數，合計至少 70／80。未接通的功能不以規格目標分代替實測分。

## 朝 80 分與正式驗收的必要修正

目前已達進入三角色評分的最低開發證據門檻。80 分仍是產品優化方向，但不得用兒童焦慮、倒數、扣收藏或假社交補分。以下是正式版與滿分目標的必要工作：

1. 社交影響：在專用後端完成班級合作 Boss、教師看板與共同修復的跨裝置真人實測，再驗證下一題是否真的採用學伴策略；不公開個人速度、錯題或排名。
2. 創意授權：加入組句卡、多解任務與學生可安全建立的策略配方，並保存實際使用證據。
3. 史詩意義：讓班級共同修復推進跨週章節，而非只增加個人數字。
4. 所有權：增加不付費、不競逐稀有度的島嶼展示配置。
5. 好奇與長期循環：增加可查明規則、無隱藏機率的跨章節探索事件與更多知識星內容。
6. 損失與逃避：救援教學機制已落地（教學卡＋既有題庫救援題＋護盾回復＋真實 `rescued` 語意）。後續為每個微技能補齊兩位教師複核過的救援題，讓「教學卡但沒有救援題」的退路徑消失；仍只採自願承諾、清楚的中斷續玩與進度保護，不用扣分、倒數、羞恥或強迫連續登入。
7. 實測：由遊戲設計、英語教學、兒童 UX 三種角色獨立評分，逐項保留畫面、行為與學生試玩證據；中位數總分未達 70 就繼續修正，不以平均分掩蓋弱項。
8. 安全：完成真實學生焦慮、羞恥、刷題副作用與家長／教師理解度測試；安全閘門未過，即使分數達 70 也不得發布。

## 兒童安全檢查

目前學生端已確認：

- 一般題沒有倒數。
- 答錯不顯示「失敗」「太慢」「你不會」或 Game Over。
- 不因錯題、缺席或中斷移除 XP、能力卡、提示、解析或必學內容。
- 沒有公開排行榜、付費優勢、限量名額、轉蛋或隱藏機率。
- 英雄光環與兩條路線只改變外觀或探索呈現，不改變題目難度、提示與 XP。
- 同桌策略接力先遮蔽內容；下一位必須主動選擇具體用法才會記錄共同修復。系統不捏造同學、教師或班級活動。
- 課堂版只呈現全班共同修復與支援狀態，不呈現個人分數、速度或排名；未連線時停在安全閘門。
- 護盾歸零進入夥伴救援教學：教學卡與救援題不計時、不扣任何收穫，救援題答錯只提示「換一個線索再試」並可繼續嘗試；救援題一律取自既有題庫，不即席生成。
- Boss 通關與能力精熟分開呈現。
- 試作題固定揭露「待兩位英語教師複核」。

尚未完成的真實學生焦慮、羞恥、刷題副作用與家長／教師理解度測試，仍屬正式發布阻擋項目。
