# 英語英雄島 Octalysis 實作稽核

更新日期：2026-07-14

## 稽核定位

本文件依周郁凱（Yu-kai Chou）的 Octalysis 八角遊戲化框架，盤點「目前已實作且可操作」的證據。這是開發階段自評，不是正式驗收分數。

正式門檻維持：遊戲設計、英語教學、兒童 UX 三種角色各自審查，各驅力取三人中位數，合計至少 70／80。未接通的功能不以規格目標分代替實測分。

## 到 70 分以上的必要修正

以下工作完成且有測試證據後，才進入三角色正式評分：

1. 社交影響：接通隱私友善的班級合作 Boss、教師預設鼓勵卡與共同修復進度；不公開個人速度、錯題或排名。
2. 創意授權：讓三種提示工具產生可觀察的策略差異，加入組句卡與兩條等值練功路線。
3. 不可預測：加入不改變題目成本的 Boss 招式、分支對白、探索事件與典藏故事。
4. 損失與逃避：完成可恢復的島嶼亮度與連續進度緩降，確認不誘發焦慮或刷題。
5. 所有權：加入安全的英雄配色、島嶼展示與已精熟能力的長期收藏。
6. 實測：由三種角色獨立評分，逐項保留畫面、行為與學生試玩證據；中位數總分未達 70 就繼續修正，不以平均分掩蓋弱項。

## 兒童安全檢查

目前學生端已確認：

- 一般題沒有倒數。
- 答錯不顯示「失敗」「太慢」「你不會」或 Game Over。
- 不因錯題、缺席或中斷移除 XP、能力卡、提示、解析或必學內容。
- 沒有公開排行榜、付費優勢、限量名額、轉蛋或隱藏機率。
- Boss 通關與能力精熟分開呈現。
- 試作題固定揭露「待兩位英語教師複核」。

尚未完成的真實學生焦慮、羞恥、刷題副作用與家長／教師理解度測試，仍屬正式發布阻擋項目。
