# 無障礙驗證與人工閘門

本文件記錄英語英雄島目前能以程式證明的範圍，以及仍須由真人與實機完成的驗收。自動化結果只能表述為「涵蓋狀態未發現 axe 可偵測的違規」，不能表述為「已通過 WCAG 2.2 AA」。

## 自動化範圍

`e2e/accessibility.spec.ts` 使用 `@axe-core/playwright`，在桌機 Chromium 與 360px 行動版 Chromium 執行下列標籤：

- `wcag2a`
- `wcag2aa`
- `wcag21a`
- `wcag21aa`
- `wcag22aa`

目前覆蓋首頁、英雄建立、診斷題與答錯救援、學生／教師／治理安全閘門，以及一條完整學生冒險的能力島、任務選擇、提示、結果、學伴策略接力、複習排程、星光秘境與能力圖鑑狀態。

每次掃描都附上完整 axe JSON。測試會阻擋所有 `violations`，也會阻擋 `serious` 或 `critical` 的 `incomplete`。唯一暫時允許的是 axe 明確標示「無法因背景漸層判定」的 `color-contrast` 結果；這些項目仍列入人工對比檢查，不能當作通過證據。

鍵盤自動化另驗證：

- Skip link 可見，啟用後焦點確實進入主要內容。
- 任務選項符合單選群組的方向鍵與 roving tabindex 行為。
- 提示與作答回饋替換畫面後，焦點不會掉回頁面開頭。

## 已納入的防護

- 主要內容可由 Skip link 聚焦。
- 動態提示與回饋提供可程式辨識的名稱與焦點位置。
- 按鈕狀態切換不使用會產生低對比中間幀的背景色動畫。
- Disabled 按鈕使用明確的前景與背景色，不以整體透明度降低可讀性。
- 自訂單選元件保留鍵盤與焦點語意。
- 聽力題提供文字輔助路徑；自主冒險與課堂 TTS 試作題使用文字輔助後都不會被記為獨立精熟。課堂端會把瀏覽器回報的輔助標記保存到離線佇列、RPC、不可變事件與匿名報表；這仍不是防竄改證據。正式 opaque 音訊尚缺受控逐字稿 reveal 路徑，完成前不得宣稱所有正式聽力題已有等值文字替代。
- `prefers-reduced-motion`、可見焦點與 360px 無橫向捲動已有樣式或流程測試。

## 正式發布前的人工閘門

下列項目尚不能由 axe 或模擬裝置取代：

1. VoiceOver＋Safari、NVDA＋Chrome／Firefox 的朗讀順序、狀態公告與焦點位置。
2. 全流程只用鍵盤操作，確認焦點可見、不被黏著元件遮住，且不會形成鍵盤陷阱。
3. 200%／400% 放大、文字間距覆寫、320 CSS px reflow、橫向與直向顯示。
4. 漸層背景、動畫中間幀、色弱模式與低亮度環境的實際文字對比。
5. 英語音訊的清晰度、語速、逐字稿正確性，以及聽障學生的等價作答路徑。
6. 圖像替代文字是否具有教學意義，且不會洩漏正確答案。
7. iOS／Android 真實觸控尺寸、誤觸、行動讀屏手勢與裝置語音合成。
8. 兒童是否能理解提示、錯誤訊息、學伴接力與秘境規則，並觀察認知負荷與挫折感。
9. 專用 Supabase 上線後的教師、治理與即時課堂完整狀態。

每一項人工閘門都應留下裝置、瀏覽器／讀屏器版本、測試日期、測試人、結果與修正紀錄；未完成前不得把本系統標示為正式 WCAG 2.2 AA 合規。
