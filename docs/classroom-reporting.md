# 課後證據與補救判讀

更新日期：2026-07-14

## 目的

課後報告協助教師決定「下一個 5–10 分鐘要怎麼教」，不是替學生貼能力標籤。報告只使用活動層級與題目層級的匿名計數，不顯示排行榜、作答速度、暱稱、個別選項或完整錯題紀錄。

## 資料來源

專用 RPC `get_activity_learning_evidence` 只允許已核准且擁有該活動的教師呼叫，回傳：

- 活動狀態、派題範圍、能力標籤與題數。
- 參與人數、有作答人數、完成全部題目人數。
- 各題作答數、獨立答對數與需要支援數。

RPC 不回傳學生暱稱、匿名使用者 ID、學習代碼、選項明細、正解、解析或題幹。原始學習事件仍維持不可修改、不可刪除。

## 保守判讀門檻

只有同時符合下列條件，系統才允許標示「可能的共通卡點」：

1. 活動狀態已完成或已結束。
2. 至少 5 位參與者。
3. 至少六成參與者完成全部題目。
4. 活動的每一題至少有 3 份作答。

任何一項不符合，結論固定為「資料不足，暫不判定共通弱點」，並列出缺少的證據。這是一道產品防呆門檻，不是正式心理計量或常模標準；未來仍須依真實班級試用結果校正。

## 判讀規則

| 結論 | 條件 | 教師介面行為 |
|---|---|---|
| 資料不足 | 任一證據門檻未達 | 不列共通弱點，建議先做 3 題等值短測補足證據 |
| 可能的共通卡點 | 證據足夠，且至少一題需要支援比例達 40% | 最多列出支援比例最高的 2 題，提供能力對應補救步驟 |
| 整體仍在發展中 | 沒有單題達 40%，但整體獨立答對率低於 80% | 建議小範圍再教，避免整班重講 |
| 表現穩定 | 沒有共通卡點，且整體獨立答對率至少 80% | 建議轉換情境與隔日抽查，不再大量刷同型題 |

百分比以實際收到的作答為分母；未作答不會直接算成答錯。介面另外呈現作答覆蓋率，讓教師看見缺席、斷線或未完成可能造成的偏差。

## 補救建議結構

每個能力的建議都遵守相同節奏：

1. 用正例／反例、句框、圖像或聲音對照，說清楚判斷線索。
2. 完成 1 題示範與 1 題口頭判斷，要求說出理由，不公布個人排名。
3. 立即派 3 題等值變式，逐步撤除提示；隔日再用 1 題新情境確認。

提示後答對與救援完成不會自動當成獨立精熟；隔日無提示答對才是較穩定的能力證據。

## 尚待驗證

- Migration 尚未套用至專用 Supabase，因此彙總 SQL 目前只有靜態契約測試，沒有真實 PostgreSQL 執行證據。
- 5 人、六成完成、每題 3 份與 40% 支援比例都是保守初始門檻，仍需英語教師與真實班級試用校正。
- 補救建議尚未經兩位英語教師逐項審查，也未做兒童學習成效實驗。
- 正式驗收需包含小組、個別、缺席、弱網路與中途結束等不同情境，確認系統不會把資料缺漏誤判為能力不足。
