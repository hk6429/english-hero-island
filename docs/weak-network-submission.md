# 弱網路作答保護

更新日期：2026-07-14

## 目前結論

課堂作答已完成「先保存在裝置、再由伺服器判分」的程式流程與自動化測試。斷線、頁面重整及伺服器已收件但瀏覽器沒收到回應時，系統會沿用同一個 `deviceEventId` 重送；文字輔助使用標記也會跟著待送答案保存。正解、解析與共同進度仍只接受伺服器回傳。

這是程式層級證據，不等於真實校園網路驗收。專用 Supabase runtime 建立後，仍須用桌機、平板與手機實測 Wi-Fi 切換、長延遲、封包中斷及多裝置同時送出。

## 資料流程

1. 學生必須先在線上取得不含正解的活動題目投影；系統不提供未下載題目的純離線作答。
2. 聽力文字輔助一旦揭露，瀏覽器先在獨立 IndexedDB 保存只含匿名題目 scope 的單調標記；它不與待送答案共用資料表，也不保存選項、題幹、逐字稿或學習代碼。
3. 點選答案後，瀏覽器先把待送紀錄寫入 IndexedDB，完成前不呼叫遠端判分；輔助標記會移交為待送紀錄的 `hintsUsed`。
4. 待送紀錄包含活動 ID、匿名參與者 ID、裝置事件 ID、選項 ID、是否使用文字輔助、時間與公開題目快照；不包含正解、解析、提示答案、姓名或電子郵件。
5. 待送期間鎖定所有選項，不允許改選或製造另一筆事件，也不在本機猜測對錯。
6. 有網路時送交 `submit_classroom_response`；離線時留在裝置，收到 `online` 事件後自動重送，也提供明確的手動重送按鈕。
7. 伺服器以同一事件 ID、作答內容與文字輔助標記維持冪等；活動結束後仍可重播完全相同的已提交事件，不同內容的衝突重送則拒絕。正確且未使用輔助才記為 `independent_correct`，正確但使用輔助則記為 `assisted_correct`。瀏覽器收到伺服器判定後，先清除待送紀錄，再顯示回饋與解析。

## 已處理的失敗情境

| 情境 | 行為 |
|---|---|
| 點選時已離線 | 只保存到裝置，不呼叫判分 RPC；恢復連線後自動送出 |
| 送出失敗 | 保留同一筆紀錄與事件 ID，顯示安全保存狀態及重送按鈕 |
| 頁面重整 | 依活動與匿名參與者找回待送紀錄，以題目快照恢復畫面 |
| 伺服器已收件但回應遺失 | 以相同事件 ID 重送，由伺服器回傳同一筆判定，不重複累積共同進度 |
| 另一位匿名參與者使用同裝置 | IndexedDB 查詢以活動 ID 與參與者 ID 分區，不讀取別人的待送紀錄 |
| IndexedDB 寫入失敗 | 阻止遠端送出並顯示警示，避免出現「已送出但本機沒有復原依據」的狀態 |

## 自動化證據

- IndexedDB 跨 store instance 恢復、同事件覆寫去重及成功後清除。
- `hintsUsed: 1` 可跨 IndexedDB store instance 恢復；舊格式音訊待送紀錄若缺少欄位，會保守視為使用輔助。
- 揭露文字後、答題前重建元件時，獨立 support marker store 仍會恢復 `hintsUsed: 1`，不同匿名參與者與題目版本互不污染。
- 不同匿名參與者的待送紀錄隔離。
- 離線時不呼叫伺服器，恢復連線後自動判分並清除佇列。
- 頁面重整且伺服器題目清單暫時為空時，仍能用不含答案的題目快照重送同一事件。

## 尚未解除的限制

- 待送資料只存在目前瀏覽器；清除網站資料、使用無痕模式結束工作階段或裝置損壞，仍可能失去尚未送達的答案。
- IndexedDB 不是跨裝置同步機制，也不是永久學習紀錄；正式紀錄仍以伺服器不可變學習事件為準。
- 文字輔助標記目前是瀏覽器回報的教學 telemetry，不是防竄改證據；正式高風險評量若需強證據，必須改成受保護的伺服器 reveal 事件。
- 文字輔助 marker 已能跨元件重建保存證據，但真實 `/join` 整頁重整仍會先失去只存在 React state 的加入畫面；完整續接需另以伺服器參與紀錄恢復匿名活動，不應把活動碼或學習代碼另存本機。
- 若瀏覽器完全不支援 IndexedDB，系統只能在目前頁面使用記憶體暫存；若 IndexedDB API 存在但寫入失敗，系統會停止送出並要求保留頁面。
- 自動化測試尚未取代真實弱網路、多人併發、瀏覽器儲存額度與無障礙狀態播報驗收。
